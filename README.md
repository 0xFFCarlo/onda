# Onda

**A tiny, low-level, stack-based language that stays fast and readable.**

Onda gives you the directness of Forth-style programming with a modern toolchain:
- bytecode compiler and VM
- optional JIT execution
- importable multi-file programs
- tiny syntax with practical control flow and locals

If you like minimal languages that still feel practical, Onda is built for that.

## Quick Start (2 minutes)

### 1. Build

```bash
make
```

This creates:
- `bin/ondac` (compiler + runner)
- `bin/tests_ondac` (test binary)

### 2. Run your first program

```bash
./bin/ondac run examples/hello_world.onda
```

Expected output:

```text
Hello World!
```

### 3. Compile to bytecode and execute

```bash
./bin/ondac build examples/hello_world.onda /tmp/hello.onbc
./bin/ondac exec /tmp/hello.onbc
```

## Why Onda

- **Minimal mental model**: everything is stack-driven and explicit
- **Low-level control**: integer ops, memory and C-style stdlib bindings
- **Fast feedback**: run source directly or execute generated bytecode
- **Performance path**: optional JIT backend
- **Small codebase**: easy to inspect, hack, and extend

## Language Taste

```onda
: fib
  if dup 2 < then
    if dup 0 == then drop 0 else drop 1 end
  else
    dup 1 - fib swap 2 - fib +
  end
;

: main 10 fib . ;
```

Run it:

```bash
./bin/ondac run examples/run_fibonacci.onda
```

## CLI

```text
ondac run   [--no-jit] [--time] [--print-bytecode] <source_file>
ondac build [--time] [--print-bytecode] <source_file> <output_bytecode>
ondac exec  [--no-jit] [--time] [--print-bytecode] <bytecode_file>
```

## Install

Install into `~/.local/bin` by default:

```bash
make install
```

This installs:
- `ondac`
- `onda` (symlink to `ondac`)

Uninstall:

```bash
make uninstall
```
